## Fixed Launch Path

```bash
export FRONTIER_PER_RANK_TRITON_CACHE=${FRONTIER_PER_RANK_TRITON_CACHE:-1}
```

The shared runner defaults the cache base only after `RUN_ROOT` is defined:

```bash
FRONTIER_PER_RANK_TRITON_CACHE=${FRONTIER_PER_RANK_TRITON_CACHE:-0}
FRONTIER_TRITON_CACHE_BASE=${FRONTIER_TRITON_CACHE_BASE:-"${RUN_ROOT}/artifacts/triton-cache"}
```

When the `srun` worker shell starts, each rank will set and create its own
cache directory:

```bash
TRITON_CACHE_DIR="${FRONTIER_TRITON_CACHE_BASE%/}/rank-${SLURM_PROCID}"
mkdir -p "$TRITON_CACHE_DIR"
```

The runner records the setting in the future run artifacts:

```text

artifacts/manifest.json: frontier_per_rank_triton_cache, frontier_triton_cache_base
```

64-node jobs have not yet started, so runtime env/manifest confirmation remains
pending until allocation.
