# E98 — E97 split-gate ON TOP of the unified capability-span + horizontal specialization

**Task:** `e98-on-e97`. Builds on
[UNIFIED_CELL_RESULTS.md](UNIFIED_CELL_RESULTS.md) (one un-clamped matrix-recurrence
kernel `S = φ(λS − β·k(kᵀS) + i·kvᵀ)` realises every capability corner),
[UNIFIED_LEARNABILITY_RESULTS.md](UNIFIED_LEARNABILITY_RESULTS.md) (a generic-init
learned cell does NOT self-organise; **init-spread + knob-LR** fixes it), and
[SPECIALIZATION_STUDY_RESULTS.md](SPECIALIZATION_STUDY_RESULTS.md) (horizontal
head-type hybridization is governed by inductive **placement**, not training
pressure; **spread-init + knob-LR** is the winning form).

E98 takes E97's improvement to the *transition* — `use_split_edit`, which
**decouples** the erase/read direction from the write — and adds it to the unified
cell, keeping ALL the unified machinery (free λ≥1, the four corners, φ
linear↔tanh↔relu, the `i` write-gate, and the spread-init + knob-LR horizontal
specialization).

The E88-based unified cell couples erase = write = `k` (`A_t = λI − β·kkᵀ`). E97
splits them: read/erase along the input-gated key `b⊙k`, write the gated value
`w⊙v`, with the outer-product (write) key the **ungated** `k`. The E98 recurrence is

> **pre_t = λ_t S − β_t · k_t ((b_t⊙k_t)ᵀ S) + i_t · k_t (w_t⊙v_t)ᵀ ;  S_t = φ_γ(pre_t)**

with `b = σ(W_b x) ∈ (0,1)^N` (erase gate) and `w = σ(W_w x) ∈ (0,1)^V` (value
gate). **`b = w = 1` reduces this EXACTLY to the E88-based unified cell** — so all
four capability corners remain reachable, and the split gate is a strict
super-set: `b<1` is an input-dependent *partial* erase the coupled cell cannot
express.

REAL training only, fp32, schedule-free AdamW, depth 4, 32 heads, N=V=32, idle-GPU
scheduler (8 GPUs). 3 seeds {42,123,456}; train T=128; eval (length extrapolation)
at T∈{128,256,512,1024}. **33/33 E98 runs completed, 0 failures.** The E88-based
columns are **reused** (not re-run) from the prior sweeps. `paper/main.typ`
untouched.

### How the corners are reached under the split-gated transition

The linear operator on `S` along the N-axis is `A_t = λI − β·k(b⊙k)ᵀ`. With
L2-normalized `k`, its eigenvalue in the `k` direction is

> **eig_along = λ − β·⟨b, k²⟩**,  where ⟨b,k²⟩ = Σₙ bₙ kₙ² ∈ [0,1].

- **b → 1 (erase open):** ⟨b,k²⟩ → 1, so eig_along → **λ − β** — the *exact*
  E88-based unified eigenvalue. Every corner the coupled cell reached is still
  reached: **track** (λ=0.9, β=1.8 ⇒ eig −0.9, reflection/S5), **count** (λ=1,
  β=0), **latch** (λ=1.3, β=0, tanh), **nonlin** (λ=0.9, β mid, tanh).
- **b < 1 (partial erase):** eig_along moves toward λ (less correction) — an
  input-dependent interpolation between "hold" and "reflect" that the coupled cell
  cannot express. This is exactly the extra freedom that buys the +0.305 on the
  track corner at T=1024.

The reflection/track corner is therefore reached the same way (β > λ with the erase
open), and the split gate adds a *per-token* dial on how much of the along-key
direction is erased.
