A recurrent language model whose state passes through a nonlinearity at every step, the property that forecloses the parallel scan linear recurrences use for throughput, reaches below one bit per byte on The Pile on a single workstation-class GPU. Throughput comes from width, not time: hundreds of small recurrent programs per token, each a bounded matrix tile, the time loop serial inside each. The measurement is held-out bits per byte; training loss is only a diagnostic. On held-out bytes a delta-correcting nonlinear model, a raw-write nonlinear model, and a linear-recurrent model are a statistical tie, so bulk language-model loss does not separate them. Two axes do. Linear against nonlinear is computability: the linear recurrence provably cannot track non-solvable group state at length, a converged ceiling, not undertraining. Delta-correcting against raw-write, both nonlinear and both in principle able, is learning efficiency: at matched budget the delta update length-generalizes far past its training length while the raw-write update under-reaches. A Lean 4 trusted core proves the delta-correcting update reaches a strictly larger one-step function class than raw-write at matched floating-point cost.

Seed: immense force held inside exact structure; angle: the two axes bulk loss hides, computability versus learning efficiency.
