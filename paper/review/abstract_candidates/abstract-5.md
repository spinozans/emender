Hundreds of small recurrent programs run inside each token, each turning its bounded matrix state through a nonlinearity at every step. That per-step nonlinearity forecloses the parallel scan linear recurrences rely on for throughput; parallelism is recovered across the width of the programs, the time loop kept serial inside each. So arranged, an attention-free recurrent model with no time-axis scan reaches below one bit per byte on The Pile on a single workstation-class GPU. We measure on held-out data and treat training loss as diagnostic only. The result is flat: a delta-correcting model, a raw-write model, both nonlinear in time, and a linear-recurrent baseline tie statistically, and bulk language-model loss does not separate them. Two axes do. Linearity sets computability: the linear recurrence provably cannot track non-solvable S5 state at length, a converged ceiling, not undertraining. The write rule sets learning efficiency: at matched budget the delta correction length-generalizes on S5 far past its training length while raw-write under-reaches. A Lean 4 trusted core proves the delta-correcting update reaches a strictly larger one-step function class than raw-write at matched FLOP.

Seed: many independent units cohering into one moving shape; angle: the width-axis multitude as the opening image, then the held-out tie that splits cleanly into a computability axis and a learning-efficiency axis.
