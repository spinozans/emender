# Reproduce the E97-RAW 1.3B LM CMA-ES Leaderboard

End-to-end recipe to regenerate the ~1.3B parameter / ctx-2k CMA-ES leaderboard
in which **`e97-raw` ranks #1** (avg loss 5.9511). Every command below cites
committed code/scripts. Results-only artifacts (~310 GB) live under the
git-ignored `experiments/local/` tree and are *not* in git; this recipe
regenerates them from a clean checkout.

- Evidence summary: [`docs/E97_RAW_1P3B_LEADERBOARD.md`](E97_RAW_1P3B_LEADERBOARD.md)
- Original handoff: [`docs/HANDOFF_CMAES_GDN2_MLP_20260605.md`](HANDOFF_CMAES_GDN2_MLP_20260605.md)

## 1. Environment

```bash
cd /home/erikg/emender                 # the durable repo (package path: ndm/)
git checkout wg/gdn2-mlp-cma-20260603  # or main after the consolidation merge
uv sync                                # project env; runner uses `uv run python ...`
# GPU: the published sweep used 8×GPU (split across models); a single 2-GPU
# pair reproduces one model at a time. Triton + bf16 kernels require a recent
# NVIDIA GPU. The external GDN-2 stack (gdn2*/fla-gdn) also needs $GDN2_PATH.
```

Key fixed settings shared by every run in this leaderboard:

- Param target: `--params 1300M`, `--param_tolerance 0.03`
- Context / chunk: `--chunk_size 2048` (ctx-2k)
- Tokenizer: `--tokenizer p50k_base`
- Data: `--data /home/erikg/elman/data/pile.txt`
- Budget per candidate: `--train_minutes 15`
- Optimizer: schedule-free AdamW (hardwired in `build_train_command`, `cmaes_search_v2.py:752`)
- CMA-ES: `--popsize 8`, anchor-seeded mean (`--anchor_configs ... --anchor_only_cmaes`)
- Env: `export CMAES_MAX_VALID_ATTEMPTS=80`

## 2. Run the sweep

The exact, verbatim launch commands are committed under
`scripts/repro_cmaes_1300m/`. Copy the anchor files into place first (the
launch scripts reference them under the local redo root):

```bash
cd /home/erikg/emender
ROOT=experiments/local/cmaes_redo_1300m_20260529      # git-ignored output root
mkdir -p "$ROOT"
cp docs/repro/cmaes_1300m_anchors/anchors_corrected.json     "$ROOT/"
cp docs/repro/cmaes_1300m_anchors/anchors_gdn2_primary.json  "$ROOT/"
cp docs/repro/cmaes_1300m_anchors/anchors_missing_20260531.json "$ROOT/"
```

### 2a. Winner + E97 family (`e97`, `e97-raw`, `e97-linear`)

```bash
bash scripts/repro_cmaes_1300m/launch_e97_queue.sh
```

This runs, per model (verbatim from the committed script):

```bash
uv run python -u scripts/cmaes_search_v2.py \
  --model e97-raw \                 # then e97, e97-linear
  --gpus 4,5 \
  --output experiments/local/cmaes_redo_1300m_20260529/e97-raw \
  --phase cmaes \
  --params 1300M --param_tolerance 0.03 \
  --train_minutes 15 \
  --popsize 8 --sigma 0.8 \
  --chunk_size 2048 \
  --tokenizer p50k_base \
  --data /home/erikg/elman/data/pile.txt \
  --min_generations 8 \
  --use_triton_e88 \
  --anchor_configs experiments/local/cmaes_redo_1300m_20260529/anchors_corrected.json \
  --anchor_only_cmaes
```

`e97-raw` is produced by the driver appending `--level E97 --e88_raw_write 1
--use_triton 1` to `train.py` (see `cmaes_search_v2.py:787-804`).

### 2b. GDN-2 baselines and remaining models

```bash
# Old mixer-only gdn2 (bookkeeping baseline)
bash scripts/repro_cmaes_1300m/launch_gdn2.sh
# e88-raw, e88-linear, fla-gdn, m2rnn (parallel, one GPU each)
bash scripts/repro_cmaes_1300m/launch_missing_cma_20260531.sh
```

### 2c. gdn2-mlp (rank 2)

`gdn2-mlp` (official-style GDN-2 + SwiGLU MLP) is launched the same way with
`--model gdn2-mlp` and `GDN2_PATH=/home/erikg/GatedDeltaNet-2`. The exact
top-up/resume invocation is preserved in
[`docs/HANDOFF_CMAES_GDN2_MLP_20260605.md`](HANDOFF_CMAES_GDN2_MLP_20260605.md)
("Recommended GDN2-MLP top-up procedure"); it requires `$GDN2_PATH`.

`e88`, `transformer` used the same `cmaes_search_v2.py` invocation pattern with
their `--model` name (no `--use_triton_e88` for non-E88/E97 models).

## 3. Aggregate the leaderboard

Once the per-model run directories contain `results.json` / `eval_*/*.done`:

```bash
cd /home/erikg/emender
python scripts/aggregate_cmaes_leaderboard.py \
  --root experiments/local/cmaes_redo_1300m_20260529
```

Expected top of the ranked output (avg-loss ranking):

```

 2 gdn2-mlp   evals= 64 complete    avg_loss=5.9613 final=5.5400 params_M=1285.2 eval=57
 3 e97        evals=109 complete    avg_loss=5.9733 final=5.5386 params_M=1274.7 eval=59
 ...
```

Use `--json` for a machine-readable dump. This script reproduces the published
table exactly from the local artifacts.

## 4. Optional: held-out BPB (paper pipeline, not the ranking metric)

The leaderboard ranks on the in-search training loss. For a separate
tokenizer-invariant held-out BPB measurement of a trained checkpoint, use the
committed paper scripts (`scripts/measure_pile_bpb.py`, `scripts/gen_bpb_report.py`,
`scripts/e88_heldout_hf_bpb.py`). These were not used to produce this ranking.
